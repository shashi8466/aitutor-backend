## ⚠️ STEP 2: Database Migration (MANUAL - You Must Do This!)

The `demo_leads` table needs to be created in your Supabase database.

### Option A: Supabase Dashboard (Easiest)

1. **Go to Supabase Dashboard:**
   ```
   https://supabase.com/dashboard
   ```

2. **Select your project**

3. **Click "SQL Editor"** (left sidebar)

4. **Click "New Query"**

5. **Copy and paste this SQL:**

```sql
/* 
# Add Demo Course Support
1. New Column in `courses` table
   - `is_demo` (boolean) - defaults to false
*/

DO $$ 
BEGIN 
    IF NOT EXISTS (SELECT 1 FROM information_schema.columns WHERE table_name = 'courses' AND column_name = 'is_demo') THEN
        ALTER TABLE courses ADD COLUMN is_demo boolean DEFAULT false;
    END IF;
END $$;

-- Create demo_leads table
CREATE TABLE IF NOT EXISTS public.demo_leads (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    course_id bigint REFERENCES public.courses(id) ON DELETE CASCADE,
    full_name text NOT NULL,
    grade text NOT NULL,
    email text NOT NULL,
    phone text NOT NULL,
    level_completed text NOT NULL,
    score_details jsonb DEFAULT '{}'::jsonb,
    created_at timestamptz DEFAULT now()
);

-- Enable RLS
ALTER TABLE public.demo_leads ENABLE ROW LEVEL SECURITY;

-- Allow public insertion for demo flow
CREATE POLICY "Public can insert demo leads" ON public.demo_leads
FOR INSERT WITH CHECK (true);

-- Allow admins to view leads
CREATE POLICY "Admins can view demo leads" ON public.demo_leads
FOR SELECT USING (
    EXISTS (
        SELECT 1 FROM profiles 
        WHERE id = auth.uid() AND role = 'admin'
    )
);
```

6. **Click "Run" button** (or press Ctrl+Enter)

7. **You should see:** "Success. No rows returned" or similar success message ✅

## 🐛 Troubleshooting

### Problem: Still getting 404 error

**Solution:**
- Wait for Render deployment to complete (check dashboard)
- Hard refresh your browser (Ctrl + Shift + R)
- Check Render logs for errors

### Problem: Getting 500 error

**Solution:**
- The database migration hasn't been applied yet
- Go to Supabase SQL Editor and run the migration SQL (Step 2)

### Problem: Email not being sent

**Solution:**
- Check if Brevo API key is set in Render environment variables
- Check Render logs for email errors
- The lead will still be saved even if email fails

### Problem: Render deployment failing

**Solution:**
- Check Render logs for build errors
- Make sure all dependencies are in package.json
- Check if there are any syntax errors in the code

## 🎯 What Happens When User Submits the Form

1. **Frontend** collects: Name, Grade, Email, Phone, Score
2. **Backend** receives POST request at `/api/demo/submit-lead`
3. **Database** saves lead to `demo_leads` table
4. **Email Service** sends score report to user's email
5. **Frontend** redirects to demo dashboard
6. **User** receives email with their predicted SAT score

**No existing features are affected** - this is completely isolated!
